Git &GitHub Study Note

Posted on 2020-03-04

What is Git?

Version Control System (VCS) for tracking changes in computer files.

Concepts of Git

  • Keep track of code history
  • Takes “snapshots of yout files”

Basic Commands

  • git init // Initialize local Git repository
  • git add // Add files to index
  • git status // Check status of working tree
  • git commit // Commit changes in index
  • git push // Push to remote repository
  • git pull // Pull repository from remote repositort
  • git clone // Clone repository into a new directory

Installing Git

  • Linux (Debian)
    • sudo apt-get install git
  • Linux (Fedora)
    • sudo yum install git
  • Mac
    • https://git-scm.com/
      or
    • brew install git
  • Windows
    • https://git-scm.com/

Create a new repository

Using Terminal, create a new directory, open it and perform a

  • git init
    to create a new git repository.

Add & commit

You can add changes (add it to the Index) using

  • git add // add single file
  • git add . // add all files under the repository
  • git add *.html // add all html files

To actually commit these changes use

  • git commit -m “Commit message”
    Now the file is committed to the HEAD, but not in remote repository yet.

  • .gitignore file
    gitignore - Specifies intentionally untracked files to ignore

Remove

You can also remove some files using

  • git rm –cached filesname

Pushing changes

First, connect your repository to a remote server, using

  • git remote add origin
    Then, send those changes to your remote repository, execute
  • git push origin master
    Change master to whatever branch you want to push your changes to.

Branching

Create a new branch and switch to it using

  • git checkout -b branch_name
    switch back to master
  • git checkout master
    and delete the branch again
  • git branch -d branch_name
    a branch is not available to others unless you push the branch to your remote repository
  • git push origin

Update & merge

Update your local repository to the newest commit, execute

  • git pull
    in your working directory to fetch and merge remote changes.
    To merge another branch into your active branch (e.g. master), use
    git merge
    Preview difference between branched by using
    git diff

Rebase

  • git rebase master branch_name
    Rebasing re-writes the project history by creating brand new commits for each commit in the original branch.

Tagging

Create a new tag named 1.0.0 by executing

  • git tag 1.0.0 1234567890
    the 1234567890 represents for the first 10 characters of the commit id.

Log

Study repository using

  • git log
    Add a lot of parameters after the commandl.
    To see only the commits of a certain author:
  • git log –author=chaolitan
    To see a very compressed log where each commit is one line:
  • git log –pretty=oneline
    Or maybe you want to see an ASCII art tree of all the branches,
    decorated with the names of tags and branches:
  • git log –graph –oneline –decorate –all
    See only which files have changed:
  • git log –name-status
    To get help,
    git log –help

Replace local changes

This replaces the changes in your working tree with the last content in HEAD.

  • git checkout – filesname

If you instead want to drop all your local changes and commits, fetch the latest history from the server and point your local master branch at it like this

  • git fetch origin
  • git reset –hard origin/master
